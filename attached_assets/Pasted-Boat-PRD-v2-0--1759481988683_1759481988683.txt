Boat — Расширенный PRD v2
0) Цели и рамки
Вертикальный маркетплейс только для катеров и водной техники (новая/б/у).
Ключевые ценности:
ИИ-публикация из минимального ввода (без выдумок).
ИИ-поиск по естественным фразам + классические фильтры и сортировки.
UX «как на Авито»: список → карточка → контакт/чат, сохранённые поиски.
Бренд: Boat, тема голубой/белый.
Неголосим: гарантий, оплаты/эскроу и доставки в MVP — позже.
1) Роли и доступ (матрица возможностей)
Гость: поиск/фильтры/просмотр карточек; «сохранить поиск» → предложить регистрацию; старт чата с «лёгкой» регистрацией.
Покупатель: избранное, сохранённые поиски, сообщения, подписки на бренды/модели/диапазоны.
Продавец-частник: публикация через ИИ, правки, архив/повторная публикация, базовая аналитика (просмотры, клики по телефону/чату).
Компания/дилер: витрина с логотипом, бейджи доверия, расширенные лимиты, промо-инструменты, сотрудники-менеджеры.
Модератор: очередь модерации, антидубликаты, жалобы, блокировки/разблокировки.
Администратор: LLM-ключи, БД-коннекты, справочники, лимиты/права, промо-слоты, метрики, аудит.
2) Информационная архитектура
Главная: большая строка поиска, быстрый выбор брендов/типов, «часто ищут», промо-блоки.
Каталог/выдача: поиск, хлебные крошки, счётчик результатов, переключатель вид плитка/список, сортировка, тумблер «сначала из моего региона», кнопка «Сохранить поиск».
Фильтры (фасеты):
Тип судна, бренд/модель, страна бренда.
Гео (регион/город/радиус).
Год, длина/ширина/осадка, пассажировместимость.
Корпус (материал), тип мотора (подвесной/стационарный), топливо, мощность/моточасы.
Состояние, прицеп в комплекте, доступность.
Тип продавца (все/частные/компании), рейтинг ≥ 4.
«Слова в описании», «Искать только в названиях», «Только с фото».
Карточка объявления: галерея, цена, CTA («Показать телефон», «Написать»), таблица характеристик, описание, карта, блок продавца, номер и дата объявления.
Профиль продавца: витрина, активные/архивные лоты, рейтинг/отзывы, бейджи («Официальный дилер» и др.), контакты.
Избранное, Сохранённые поиски, Сообщения/чат.
Админ-панель (см. §7).
3) Публикация через ИИ (минимум фантазии)
3.1 Минимальная форма продавца
Обязательные: бренд, модель (если знает), год (если знает), длина (м/ft), тип судна, тип мотора (если знает), состояние, локация, цена, свободный текст. + Фото/видео (1…N).
3.2 Пайплайн «Draft → Review → Publish»
Нормализация: приведение единиц (м/ft, л.с./kW), правка бренда/модели по справочнику, извлечение структурированных фактов из текста/EXIF/подписей.
Обогащение фактами (grounding): попытка подтянуть табличные характеристики (длина/ширина/осадка/пассажиры/корпус/тип мотора/бак и т. п.) из открытых источников/локального справочника.
Не найдено надёжно → «не указано» (строго без домыслов).
Для найденного — сохраняем ссылку-источник (в данных).
Генерация описания (структура):
«Ключевые параметры» → «Состояние и обслуживание» → «Комплектация/опции» → «Что важно знать».
Тон нейтральный, без маркетинговых фантазий, язык — авто по вводу.
Анти-дубликат: поиск по хэшам фото, VIN/серийным, текстовым шинглам.
Модерация: стоп-контент, контакты в тексте, неверная категория; при рисках — ручная проверка.
Экран подтверждения: показываем, что ввёл пользователь / что добавил ИИ; редактирование; подтверждение.
Публикация: статусы draft/moderation/published/blocked/sold.
3.3 Требования к ИИ
«Минимально выдумывать»: нет факта — нет значения.
Прозрачность: помечаем авто-поля и источники.
Локализация RU/EN по необходимости.
4) ИИ-поиск и ранжирование
4.1 Интерпретация запроса (NL → фильтры)
Примеры:
«старый катер до 10 м с каютой в Краснодаре» → {year≤X?, length_m≤10, category=каютный, geo=КК}.
«sea ray от 2015, не старше 300 м/ч» → {brand=Sea Ray, year≥2015, engine_hours≤300}.
Модель возвращает filters + чипы, показываем пользователю; неясности → уточняющие вопросы.
4.2 Классика каталога
Сохранённые поиски + уведомления (email/web-push/мессенджер).
Сортировка: по дате, по цене, по близости (если «сначала из региона»).
Кнопка «Показать N объявлений» после изменения фильтров.
4.3 Правила ранжирования
Отбор по фильтрам.
Скоринг: текстовая релевантность + гео-приоритет (если включён) + репутация продавца (рейтинг/отклики/доля ответов) + новизна + качество карточки (полнота/фото) + промо-бусты.
Промо-флаги «продвинуто/закреплено/выделено» поднимают в рамках допустимых ограничений.

Поведение: CTA «Телефон/Написать» закреплены при скролле.
Прозрачность: отметить, какие поля «указал продавец», какие «добавлены автоматически (ИИ)».
Мобайл: галерея сверху, CTA крупными кнопками, затем таблица → описание → карта.
7) Продавцы, доверие и монетизация
Типы продавцов: частник, компания/дилер (верификация документов для бейджа).
Рейтинг и отзывы: 1 отзыв = 1 реальное взаимодействие; антинакрутка/жалобы.
Промо: поднятие, выделение, закрепление (флаги в данных + влияние на ранжирование).
Лимиты размещений: управляются в админке по роли/региону/категории.
8) Коммуникации и лид-ивенты
Показ телефона: фиксируем событие (для аналитики/биллинга промо).
Чат: сообщения/вложения, статус прочтения, анти-спам; быстрые шаблоны вопросов («Актуально?», «Где посмотреть?»).
Уведомления: новые сообщения, ответы продавца, изменения цены, новые лоты по сохранённому поиску (частота/типы — настраиваются).
9) Модерация и анти-фрод
Правила: запрет контактов и ссылок в тексте, корректная категория, требования к фото.
Анти-дубликаты: перцептивные хэши фото, шинглы текста, VIN/серийные; политика склейки/блокировок.
Риски: стоп-слова/паттерны, массовая заливка, подозрительная активность; авто-очередь на ручную проверку.
Жалобы: типы причин, SLA обработки, авто-скрытие при большом числе жалоб.
10) Админ-панель (важно)
Ключи ИИ-провайдеров: добавить/редактировать, приоритеты, лимиты на роль/организацию, ротация, лог расхода.
Подключения к БД: создать/править, тест соединения, выбор рабочих таблиц/представлений, просмотр схем (только метаданные).
Справочники: бренды/модели/синонимы; единицы и конвертеры (м↔ft, л.с.↔kW); «часто ищут».
Модерация: стоп-слова, авто-правила, очередь ручной проверки, шаблоны ответов, журналы решений.
Анти-дубликат: отчёты по совпадениям, настройки порогов.
Лимиты/права/промо: роли, пакетные размещения, цены, расписание слотов.
Отчёты/метрики: публикации/день, конверсия (листинг→карточка, карточка→контакт), CTR чата/телефона, жалобы, время модерации.
Аудит-лог: кто/что изменил (ключи, правила, блокировки).
11) Данные (логическая модель, без тех.)
listing: id, seller_id, title, price, currency, location_id, status{draft/moderation/published/blocked/sold}, promoted_flags[], created_at, updated_at.
boat_specs (1–1): brand, model, year, length_m, beam_m, draft_m, hull_material, engine_type, hp, fuel, seats, passengers, engine_hours, trailer_included, condition, country_of_brand, availability, source_refs[].
media: id, listing_id, kind{image/video}, url, order, perceptual_hash.
seller: id, type{private/company}, name, rating, reviews_count, badges[], join_date, response_rate.
review: id, seller_id, author_id, rating, text, created_at.
conversation/message: buyer_id, seller_id, listing_id, content, attachments[], read_flags, timestamps.
saved_search: id, user_id, query_json (фильтры + исходная NL-фраза), notify_channel, cadence.
favorite: user_id, listing_id.
moderation_event: id, listing_id, reason, status, moderator_id/model_tag, created_at.
fraud_signal: id, subject_type{listing/seller}, subject_id, kind, score, created_at.
promo_order: id, seller_id, listing_id, promo_type, start_at, end_at.
audit_log: id, actor_id, action, entity, before/after, created_at.
12) UX-детали и стандарты
Светлая тема; бренд-цвет — голубой; акценты/кнопки — в одном стиле.
Плитка/список: по умолчанию — список (выше конверсия в контакт), плитка — для визуального серфинга.
Карта в карточке — приблизительная точка (privacy).
Статусы и бейджи: «Официальный дилер», «Проверено», «Продвинуто» — видны на карточке и в выдаче.
Доступность: контраст, alt-тексты, навигация с клавиатуры.
Мобайл-первый: все критичные CTA — крупные и «липкие».
13) Масштаб и качество (без тех.)
Производительность: TTI до ~2–3 с на медленном мобильном; время ответа поиска/фильтров ≤ ~1 с (p95); раскрытие фото — мгновенно.
Витринность: выдерживать десятки тысяч активных лотов, сотни тысяч просмотров/день; стабильность чата.
Надёжность: не терять черновики; автосейв при публикации; все модерационные действия — в журнале.
Приватность: маскирование телефонов; явные согласия на коммуникации; экспорт личных данных по запросу.
14) Метрики продукта
Поиск: CTR топ-3, частота уточнений, доля «пустых» выдач.
Карточка: CTR «Показать телефон» / «Написать», глубина просмотра галереи, скролл до таблицы/описания.
Публикация: доля авто-заполненных полей, % «модерация прошла с первого раза», время до публикации.
Качество: доля жалоб/блокировок, повторные контакты по одному лоту.
Монетизация: доход по промо-слотам, LTV дилера.
